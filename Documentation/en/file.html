File manager

The file manager allows to load/save individual Elody files in several format. When saving, the file format must be chosen using the pop-up menu. Another way of saving/loading Elody objects is by using the Document window. The Fetch button allows to download Elody files or MIDI files directly using an URL.

HTML format

Elody documents are saved in HTML format. It means that Elody musical objects can be visualized in a browser like for example Mozilla Firefox. The .html extension must be explicitly set so that Elody can later find and load the files.

MIDIFile format

Elody can load and save MIDIFiles format 0 and 1. When these MIDIFiles are loaded in Elody, only note events are used. Other MIDI events (like controlers, programs change, etc.) are currently ignored. Elody documents are saved in MIDIFile format 0 (one track). The .mid extension must be explicitly set so that Elody can later find and load the files.

GUIDO format

The GUIDO Music Notation Format is a formal language for score level music representation. It is a plain-text, i.e. readable and platform independent format capable of representing all the informations contained in conventional musical scores. The GUIDO Music Notation Format is developed by Holger H. Hoos and his team at the Darmstadt University of Technology, Germany.

Elody can load and save document in Basic GUIDO format.

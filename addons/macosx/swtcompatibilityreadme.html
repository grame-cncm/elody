SWT Compatibility Libraries for J2SE 5.0 Release 4

May 16, 2006

The SWT Compatibility Libraries address issues in J2SE 5.0 Release 4 to make it compatible with the Mac OS X implementation of the SWT_AWT bridge in Eclipse 3.2RC3 and later. See the Eclipse Javadoc for details on how to use this library in your SWT application. The SWT Compatibility Libraries do not replace any existing installation of Java.

Note: This build has received only limited testing and should not be installed on a system with critical data.

To Install this release:

System Requirements

SWT Compatibility Libraries for J2SE 5.0 Release 4 requires:

  • An Intel or PowerPC-based Macintosh
  • Mac OS X v10.4.5 or later
  • J2SE 5.0 Release 4

Installation Instructions

Double-click "J2SE50Release4SWTCompatibility.dmg" to mount the image on your desktop. Once the correct image has mounted, double-click the "J2SE50Release4SWTCompatibility.pkg" installer in the mounted image and follow the installation instructions.

How to determine if you have this build installed:

This package consists of two files: 

/System/Library/Frameworks/JavaVM.framework/Versions/1.5.0/Home/lib/ext/frameembedding.jar
/System/Library/Frameworks/JavaVM.framework/Versions/1.5.0/Libraries/libembedding.jnilib

Both files are necessary for the SWT_AWT bridge to work.


Major bugs fixed in this release:

  • AWT frames can be embedded in HIViews via the SWT_AWT bridge in Eclipse 3.2RC3 and later
Known problems in this release:
  • System.exit() may be needed after the SWT main event loop exits.
  • Embedded AWT Frames cannot overlap SWT widgets or other embedded frames.
  • AWT exceptions thrown from inside a native method may crash the application.
  • Directly manipulating AWT or Swing UI objects or using SwingUtilities.invokeAndWait(Runnable) results in deadlock if used from the SWT display thread.
  • SWT objects cannot be embedded in AWT windows.
Areas to Test:
  • Test any Eclipse plugins that you know rely on SWT_AWT support (http://www.eclipseplugincentral.com/)
  • Test your SWT applications that need to embed or otherwise use the AWT.


Reporting Bugs:

Please report any bugs not mentioned in the "known problems" section above. Bugs not reported via the Bugreporter aren't in the bug database and cannot be tracked and fixed.

  • Use the bug reporter on the Developer Connection website at: http://developer.apple.com/bugreporter/
  • Please include a test case if possible. A small, reproduceable test case that can be run from the command line is the most valuable piece of feedback. If your test can be expressed in the JUnit format (http://www.junit.org/) and can reproduce and detect the bug without human intervention, your JUnit test may be added to Apple's regression testing suites.
  • Please put "SWT_AWT:" at the beginning of your bug title.
  • For crashing bugs, include both the native and the Java stack crawls:
    • The native crash log is located in the standard location for CrashReporter, ~/Library/Logs/CrashReporter/.
    • The Java-specific crash information is located at: ~/Library/Logs/Java/JavaNativeCrash_pidXXXX.log where XXXX is the process ID of the crashing process.
  • For hangs/deadlocks, perform 'kill -QUIT' on the process ID and copy the resulting data from Console, stdout in the Terminal, or in the Eclipse console. Also, please call 'sample XXXX 3', where XXXX is the process ID of the hanging process (and 3 represents the duration of the sampling period in seconds).
  • If possible, check the behavior using the SWT and/or Eclipse 3.2 on another platform. If the bug occurs in both versions, please note that in your bug report.
